# Architecture Decision Records (ADR)

Ce dossier contient les decisions architecturales du projet **{{PROJECT_NAME}}**.

## Format

Chaque ADR suit le format **MADR** (Markdown Any Decision Records) :

```markdown
# ADR-NNN : Titre de la decision

## Statut
Accepte / Propose / Remplace par ADR-XXX (date)

## Contexte
Pourquoi cette decision est necessaire.

## Decision
Ce qui a ete decide.

## Alternatives considerees
| Critere | Option A | Option B (choisie) |
|---------|----------|---------------------|
| ...     | ...      | ...                 |

## Consequences
- (+) Avantages
- (-) Inconvenients
```

## Convention de nommage

`ADR-NNN-titre-court.md` — numerotation sequentielle (ADR-001, ADR-002, etc.)

## Index

| ADR | Titre | Statut | Date |
|-----|-------|--------|------|

> Les ADR sont des instantanes — on ne modifie pas un ADR existant, on en cree un nouveau qui le remplace.
